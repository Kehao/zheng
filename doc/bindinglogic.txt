角色
  银行客户经理 User
  客服       CS
  开发者后台   Admin

故事1：
  User
    上传     一批公司列表（可能包含部分水电信息）
    查看     这批公司                  信息获取情况
    查看     这批公司                  信息获取内容
    修改     某些公司                  信息查询关键词，
    提交     某些公司                  信息内容不准确反馈
    提交     某些公司                  重新获取信息内容

  CS
    查看     某个User                 新导入的公司列表
    查看     一批公司                  导入信息
    添加     一批公司                  水电账号信息
    确认     一批公司                  水电信息获取来源
    确认     一批公司（审核）            水电信息
    添加     某些公司                  水电信息
    提交     某些公司                  信息内容不准确反馈
    提交     某些公司                  重新获取信息内容

  Admin
    查看     所有公司                  信息请客
    查看     某些公司                  业务反馈

main_logic:
  1. 银行用户 根据我们提供的xls_template, 上传N个公司列表
  2. 上传完成后， 系统显示，所有导入公司，公司基本信息和公司信息获取情况
  3. 客服人员，登陆系统后，能看到当前银行用户列表，和他们各自导入的公司列表
  4. 客服人员，依据公司列表，给每个公司添加、核实相关的水电账户信息
  5. 系统根据公司水电账户信息，自动分配爬虫，接口，并获取信息。
  6. 客服人员，将看到爬取的水电信息列表，并抽样审核通过。
  7. 客服人员，将查看一批需要手工查询，添加水电信息的公司列表
  8. 客户人员，根据这批公司和水电信息，手动添加水电信息。
  9. 客服人员，将查看到一批手工添加的水电信息列表，并抽样审核通过。
  10. 银行用户，可以查看最新通过审核的水电信息 和 异常情况。
