角色：
  银行客户经理

STORY:
  银行客户经理  导出  当前状态下所有的企业列表（包括企业信息，法人信息，是否有法务异常）
  银行客户经理  导出  法务异常的企业列表（包括企业信息，法人信息，对应的法务异常信息）- 用于统计
  银行客户经理  导出  与上次导出不同的法务信息列表（包括：新添加企业的法务信息，新发现的法务信息，更新了的信息）*主要



舆情API基本需求：
模型相关(base modal):
  ITEM: 关注对象
    监控企业的模型，包含：企业名称，关联标签等，例：
      :item => {:name => "阿里巴巴集团有限公司", :tags => ["马云", "阿里巴巴", "电子商务"， "杭州"]}

  INFO: 舆情信息
    一条舆情信息，包含：对应的url，标题，发布时间，摘要，关键词，关联词，对应items，发现时间等；
      

关于关注对象：(制定危机预警方案)
  写入接口：
    post    ->  一个或多个需要关注的item，返回一个或一组ID
                JSON/XML格式的一组企业信息，包括（企业名称，及其相关额外tag（法人姓名，所属行业，地区等））

    post    ->  一组舆情关键词，作为检测舆情结果的权重衡量

    post    ->  更新某个需要关注的item，其他相关信息，更新某个关键词

    delete  ->  删除某个关注度的item，删除某个关键词

  读取接口：
    get     ->  获取当前监控的所有item列表，及其相关预警方案信息;
                JSON/XML格式的一组数据，包括建立时间，相关检索结果统计等额外数据；

    get     ->  获取某一组监控item的列表数据; 根据赛选参数，返回对应结果列表；

    get     ->  某个监控item的方案信息，及其相关tag，及其更多的语言分析后关注item；
                包括（公司名称，相关简称，相关标签，其对应的舆情关键词，包括详细预警信息；


   
关于监控结果：(事态发展)
    get     ->  获取当前已所有监控对象item的舆情结果列表
                可以根据某一个条件筛选（是否存在异常舆情，保护某一tag，对应某一舆情关键词等）

    get     ->  根据ID，获取某一个关注对象（item）的所有舆情结果；

关于同步channel通信：(传递和沟通信息)
    get     ->  系统运行状态
    
    get     ->  当前ITEM数量和舆情统计情况

    get     ->  获取一组最新时间段的舆情信息列表（尚未同步push过的舆情预警发现）
                包含时间段区间说明。

基本流程：
  a) 通过写入接口，添加一批企业列表（包括企业名称，关联的：法人，行业，地区，上下游等其他tags）；
  b) 添加一批带有权重的关键词；
  c) 查看舆情预警设置的列表矩阵，并更新调整；
  d) 开始工作，获取并同步推送最新发现的舆情；
  e) 通过查询接口，查询全部发现的舆情结果和相关统计信息；
